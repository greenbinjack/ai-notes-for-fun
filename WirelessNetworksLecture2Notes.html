Overview & Learning Goals

What Lecture 2 teaches and why it matters for your entire course

What This Lecture Is About

Lecture 2 steps back from the physics of Lecture 1 to survey the entire landscape of wireless access technologies. While Lecture 1 asked "why is wireless hard?", Lecture 2 asks "what systems have engineers built despite those difficulties?"

The word "access network" is precise: it refers to the portion of a network that connects end-user devices to the broader network infrastructure (the internet, a corporate LAN, a PSTN). Access networks are the "last mile" — the final hop between a fixed infrastructure and a mobile or portable device.

Mental Model

Think of the internet as a highway system. The core network (backbone) is like interstate highways. The access network is your local street — the part that actually connects your home to the highway. Wireless access networks are wireless versions of that last connection.

Core Learning Outcomes:

  • Identify and distinguish all major wireless access network types (cellular, WLAN, PAN, WiMAX, satellite, ad-hoc)
  • Understand cellular architecture: cells, BSS, MSC, handoff, frequency reuse
  • Understand 802.11 (Wi-Fi) architecture and MAC protocol at a functional level
  • Compare technologies by coverage, mobility, data rate, and use case
  • Trace the evolution from 1G to 5G cellular

What is a Wireless Access Network?

Defining the boundary between infrastructure and user devices

The Access Network Defined

A wireless access network is the radio-based infrastructure that enables end-user wireless devices (phones, laptops, IoT sensors, cars) to connect to a fixed network infrastructure. It is defined by two boundaries:

Air Interface Side (User-facing)

The radio link between the user device and the first fixed network node (base station, access point, satellite). Governed by a wireless standard (802.11, LTE, 5G NR).

Network Side (Infrastructure-facing)

The connection between the first fixed node and the core network (internet gateway, mobile switching center). Usually wired (fiber, copper) or microwave backhaul.

Key Entities in Any Wireless Access Network:

Entity Role Examples
Mobile Station (MS) End-user device that communicates wirelessly Smartphone, laptop, IoT sensor, car
Base Station (BS) / Access Point (AP) Fixed radio node that manages wireless connections Cell tower, Wi-Fi router, femtocell
Controller Manages multiple BSs, handles handoffs, resource allocation BSC (2G), RNC (3G), MME (4G), Wi-Fi Controller
Core Network Gateway Connects access network to internet/PSTN MSC (cellular), Router (Wi-Fi), GGSN (GPRS)

Infrastructure-Based vs. Ad-Hoc Networks

This is a fundamental classification that cuts across all wireless systems:

Infrastructure-Based

Requires fixed infrastructure (base stations, access points). All communication goes through the infrastructure node. The BS/AP manages access, coordinates transmissions, and connects to the broader network.

Examples: Cellular (2G/3G/4G/5G), Wi-Fi (infrastructure mode), WiMAX, satellite

Ad-Hoc (Infrastructure-Free)

Devices communicate directly with each other without a central coordinator. Each device can route traffic for others. No fixed infrastructure needed — the network is the devices themselves.

Examples: Wi-Fi Direct, Bluetooth piconet, VANET, military tactical networks, disaster recovery networks

Critical Distinction for Exams

802.11 (Wi-Fi) supports BOTH modes: Infrastructure mode (AP present, devices connect through AP) and IBSS/Ad-hoc mode (devices connect directly without AP). Your professor may ask which mode uses BSSID vs IBSSID, or when you'd choose ad-hoc. Also: Bluetooth can be infrastructure-less (piconet with one master, up to 7 slaves).

Professor's Picky Corner — Access Networks
Is the Wi-Fi access point part of the "access network" or the "core network"? The AP itself (its radio interface) is part of the access network. The AP's uplink (Ethernet to the router/ISP) is the boundary. The router and beyond is the core/internet. This boundary matters for understanding where QoS is applied and where latency is introduced.
An ad-hoc network has no base station. Who controls channel access? Each device runs the same MAC protocol (CSMA/CA in 802.11 IBSS mode). There is no central coordinator — distributed coordination. This is different from infrastructure mode where the AP can use PCF (Point Coordination Function) for centralized polling.

Types of Wireless Networks

The full taxonomy from body-area to global satellite networks

Classification by Scale

Wireless networks are classified by geographic coverage area. Each scale has a corresponding standard, typical data rate, and design philosophy. The slide presents this taxonomy — know all tiers deeply.

Network Type Acronym Coverage Typical Range Standard(s) Primary Use
Body Area Network BAN/WBAN On/around body ~1–2 m IEEE 802.15.6, Bluetooth LE Medical sensors, smartwatches, fitness trackers
Personal Area Network PAN/WPAN Personal space ~10 m IEEE 802.15.1 (Bluetooth), 802.15.4 (Zigbee) Headsets, keyboards, IoT home sensors
Local Area Network LAN/WLAN Building/campus ~100 m IEEE 802.11 (Wi-Fi) Internet access in homes, offices, campuses
Metropolitan Area Network MAN/WMAN City-wide ~50 km IEEE 802.16 (WiMAX) Broadband for underserved areas, enterprise backhaul
Wide Area Network WAN/WWAN National/global Cellular: up to 35 km per cell 2G/3G/4G LTE/5G NR Mobile voice, mobile internet, anything everywhere
Global Area Network GAN Global Anywhere on Earth Satellite (Iridium, Starlink, Inmarsat) Remote sensing, aviation, maritime comms, rural internet

Deep Dive: Why Each Scale Needs a Different Technology

It's not just range that distinguishes these categories — it's a combination of energy, mobility support, data rate, latency, and infrastructure requirements:

Energy: Medical body sensors (ECG monitors, glucose sensors, pacemakers) must run for years on tiny batteries. Wi-Fi radios consume 50-200 mW during transmission. Bluetooth LE consumes as little as 0.01-0.5 mW in connected mode. This 100-1000× difference is decisive.

Latency: Body area medical sensors often need guaranteed low-latency delivery (e.g., a drug pump controller must respond in milliseconds). 802.15.6 (WBAN standard) is designed for medical-grade reliability with TDMA slots ensuring deterministic access.

Interference: The human body absorbs and scatters microwave signals. At 2.4 GHz (Bluetooth) and especially 60 GHz (mmWave), the body is a significant obstacle. WBAN designs account for on-body channel models (very different from indoor propagation models).

Cost: Wi-Fi operates in unlicensed spectrum (ISM bands) — no license fee. Cellular spectrum licenses cost billions. A Wi-Fi AP costs $50-200; a cellular base station costs $100,000+.

Data Rate: 802.11ax (Wi-Fi 6) achieves up to 9.6 Gbps theoretical. LTE-Advanced peaks at ~1 Gbps, 5G at ~20 Gbps (but shared among many users). For indoor dense environments, Wi-Fi wins on raw throughput per device.

Mobility: Wi-Fi is not designed for handoff at vehicular speeds. Cellular is. If you're driving at 100 km/h, cellular manages seamless handoffs between towers every few minutes. Wi-Fi would drop connection each time you left AP coverage.

Control: You own and control your Wi-Fi network. Cellular is controlled by the operator. Enterprise IT prefers Wi-Fi for security policies, traffic inspection, and local network integration.

Ubiquitous Coverage: Cellular operators deploy towers to cover entire countries. A user moving from city to countryside to another city remains connected. No other technology achieves this at scale.

Mobility Management: The cellular core network (HLR/HSS/AMF) tracks user location as they roam between thousands of cells. When someone calls you, the network knows which cell you're currently in and routes the call there. This location tracking and routing is sophisticated — Wi-Fi, WiMAX, etc. don't provide this at national scale.

Quality of Service Guarantees: Cellular operators can provision QoS classes — voice gets priority over data, emergency services get dedicated capacity. The network core enforces these policies centrally.

Billing/Authentication: SIM card-based authentication (USIM in LTE/5G) provides strong identity management, enabling per-user billing, roaming agreements, and lawful intercept.

Professor's Picky Corner — Network Types
The slide mentions IEEE 802.15 for PANs. But IEEE 802.15 has multiple sub-standards. What are they? 802.15.1 = Bluetooth (2.4 GHz, up to ~3 Mbps); 802.15.3 = High-rate WPAN (UWB); 802.15.4 = Low-rate WPAN (ZigBee, 250 kbps, very low power, mesh capable); 802.15.6 = WBAN (body area). Your professor may ask specifically about 802.15.4 vs 802.15.1.
WiMAX (IEEE 802.16) is sometimes called a "4th generation" pre-cursor. Is it 4G? The ITU defined IMT-Advanced requirements for 4G. WiMAX (802.16e) met these requirements and was approved as "4G" alongside LTE-Advanced. However, commercially, "4G" became synonymous with LTE. WiMAX was largely displaced by LTE in the market, though 802.16 is technically a valid 4G standard.

Why You Cannot Have Everything

The diagram reveals a fundamental tradeoff: technologies that cover large areas must use lower frequencies (better penetration and range) with smaller channel bandwidths (limited spectrum at low frequencies) → lower data rates. Technologies that achieve high data rates use wider channels at higher frequencies → shorter range. High mobility requires Doppler compensation overhead → reduces effective data rate. These are physics constraints, not engineering failures.

Cellular has 30+ years of standards evolution, global roaming agreements, SIM-based identity management, and carrier-grade reliability. This is the "macro + small cell" heterogeneous network (HetNet) approach.

Wireless Technologies Deep Overview

Standards, frequencies, and architectures at a glance

The Standards Landscape

The IEEE (Institute of Electrical and Electronics Engineers) produces the 802.xx family of standards for networking. The 802.11/15/16 sub-families cover wireless. Understanding the numbering convention is essential:

IEEE 802 Numbering Convention

802.XX.Y where XX is the working group (11 = WLAN, 15 = WPAN, 16 = WMAN) and Y is the amendment letter. The base standard is the foundation; amendments add capabilities. 802.11ax is Amendment ax to the 802.11 base standard. The Wi-Fi Alliance brands these as "Wi-Fi 6" (802.11ax), "Wi-Fi 5" (802.11ac), etc. for consumer clarity.

IEEE 802 wireless standards family — three working groups covering WLAN, WPAN, and WMAN
Professor's Picky Corner — Standards
What is the difference between IEEE 802.11n and Wi-Fi 4? They're the same thing. The Wi-Fi Alliance created consumer branding (Wi-Fi 4, 5, 6, 7) to map to IEEE standards (802.11n, ac, ax, be). But IEEE 802.11p (VANET) has no Wi-Fi Alliance branding — it's not "Wi-Fi" in the consumer sense.
The slide mentions Bluetooth is under IEEE 802.15.1. Is all Bluetooth covered by 802.15.1? No! 802.15.1 standardized Bluetooth Classic (BR/EDR). Bluetooth Low Energy (BLE/Bluetooth LE) is defined by the Bluetooth SIG (Special Interest Group) in the Bluetooth Core Specification, not IEEE. BLE is not formally an IEEE standard, though 802.15.4 (Zigbee) has some overlap in the IoT space. This distinction matters.

Cellular Architecture — Complete Deep Dive

The cell concept, frequency reuse, and the full network architecture — the most math-heavy topic in Lecture 2

Why "Cellular"? The Core Insight

Before cellular, mobile telephone systems used a single powerful transmitter covering an entire city from one antenna (e.g., AT&T's IMTS system in the 1960s-70s). This had fundamental limitations:

  • Frequency reuse impossible: One transmitter used the spectrum for the entire city — no one else could use the same frequencies → very limited capacity.
  • Power vs. range: To cover a city, you needed enormous transmit power. This drained mobile device batteries (or required very heavy batteries).
  • Limited users: IMTS (Improved Mobile Telephone Service) supported only 44 channels nationally in the US. A city of millions shared 44 channels.
The Cellular Revolution — Bell Labs, 1947/1970s

D.H. Ring at Bell Labs proposed the cellular concept in 1947: divide the coverage area into small "cells," each served by a low-power base station. Adjacent cells use different frequencies, but non-adjacent cells can reuse the same frequencies. This means the same spectrum can be reused across the city simultaneously, multiplying capacity by the number of cell clusters. Bell Labs engineers reenergized this concept in the 1970s, leading to the first commercial cellular deployment (1G AMPS in 1983).

The Cellular Network Architecture Components

GSM/2G/3G cellular network hierarchy: MS ↔ BS ↔ BSC ↔ MSC ↔ PSTN/Internet. LTE/4G replaces this with eNB (evolved Node B) connected directly to EPC (Evolved Packet Core) via the S1 interface — no BSC layer.

Component Roles — Detailed Breakdown:

Mobile Station (MS) / User Equipment (UE)

The user's device. Contains: (1) Mobile Terminal (MT) — the radio hardware, (2) Terminal Equipment (TE) — the device itself (phone, laptop), (3) USIM/SIM card — stores user identity (IMSI), encryption keys, subscriber info. The SIM is the "identity" — the MT is just a radio. You can move your SIM from one phone to another and keep your number/service.

Base Transceiver Station (BTS) / Node B / eNB / gNB

The radio tower. Handles: (1) Radio transmission and reception on assigned frequencies, (2) Modulation/demodulation, encoding/decoding, (3) Power control of mobile devices, (4) Timing synchronization. One BS serves one cell. In LTE (4G) and 5G, the eNB/gNB also handles more intelligence (handoff decisions moved from BSC to eNB in LTE's "flat architecture").

Base Station Controller (BSC) / Radio Network Controller (RNC)

Manages 10-100 base stations. Handles: (1) Radio resource management (allocates channels to users), (2) Handoff decisions (in 2G/3G; moved to eNB in 4G), (3) Power control coordination, (4) Statistical multiplexing of multiple BS traffic onto fewer backhaul circuits. In LTE (4G), the BSC is eliminated — its functions are either moved to the eNB (handoff) or the core network (resource management).

Mobile Switching Center (MSC) + Databases (HLR/VLR)

The brain of the cellular network. MSC: routes calls, manages connections between cellular and PSTN, handles billing. HLR (Home Location Register): permanent database of all subscribers — their services, call forwarding settings, current VLR. VLR (Visitor Location Register): temporary database in each MSC area — stores info about roaming subscribers currently in this area. When you roam internationally, the visited MSC's VLR queries your home HLR to verify identity and get your service profile.

Professor's Picky Corner — Cellular Architecture
  • In GSM (2G): MS → BTS → BSC → MSC. In LTE (4G): UE → eNB → EPC (MME + S-GW + P-GW). In 5G NR: UE → gNB → 5GC (AMF + SMF + UPF). Your professor may ask to identify which generation a specific architecture belongs to.
  • The HLR/VLR distinction is frequently tested: HLR = permanent home database (one per operator or region); VLR = temporary visiting database (one per MSC area). When you travel, your VLR query to HLR is what enables roaming authentication.
Why does 4G LTE eliminate the BSC? LTE uses a "flat architecture" — intelligence is distributed to the eNBs (which communicate via the X2 interface) rather than centralized in a BSC. This reduces latency (no BSC hop), enables faster handoff, and reduces network cost. The tradeoff is that eNBs are more complex than 2G/3G BSs.

Cell Splitting & Frequency Reuse — The Math

The fundamental capacity equations that drive cellular network planning

The Hexagonal Cell Model

Real cellular coverage areas are irregular (terrain, buildings, interference). For planning purposes, cells are modeled as hexagons. Why hexagons, not circles or squares?

  • Circles — would overlap (gaps between circles) or leave uncovered areas if packed. Cannot tile a plane without gaps.
  • Squares — tile perfectly but corner cells are farther from the BS than edge cells, creating uneven coverage.
  • Hexagons — tile perfectly, and the distance from center to any edge is uniform (all 6 edge midpoints equidistant). This gives the most uniform coverage with the smallest number of cells. Hexagons are the closest approximation to circles that can tile a plane.
N=7 hexagonal frequency reuse cluster. Each F1–F7 represents a different frequency set. Adjacent cells always use different frequencies. The minimum distance before reusing F1 is D = R√(3N).

Frequency Reuse Mathematics — Complete Derivation

Key Variables
N
Cluster size = number of cells per cluster. Neighboring clusters reuse the same frequencies. Valid N = 1, 3, 4, 7, 9, 12, 13, 19... (only values satisfying N = i² + ij + j² for integers i, j ≥ 0)
R
Cell radius — distance from BS at center to cell edge (corner of hexagon)
D
Frequency reuse distance — distance between centers of two cells using the same frequency
S
Total available spectrum (bandwidth) for the system
k
Number of channels per cell = S / N
Formula 1: Reuse Distance
D = R · √(3N)
→
Derived from hexagonal geometry: moving from one F1 cell to the nearest other F1 cell requires crossing N cell-diameters
→
Equivalently: D/R = √(3N) — the "reuse ratio"
Formula 2: Valid Cluster Sizes
N = i² + ij + j²
→
i and j are non-negative integers (not both zero)
→
Valid N: 1 (i=1,j=0), 3 (i=1,j=1), 4 (i=2,j=0), 7 (i=2,j=1), 9 (i=3,j=0), 12 (i=2,j=2), 13 (i=3,j=1), 19 (i=3,j=2)...
→
N=4 and N=7 are the most common in practice (analog cellular used N=7; GSM used N=4 or N=7)
Formula 3: Capacity (Channels per Cell)
k = S / N
→
Larger N → more cells before reuse → more separation → less co-channel interference → better quality
→
But larger N → fewer channels per cell → less capacity per cell
→
This is the fundamental tradeoff: capacity vs. quality (interference protection)
📐 Worked Example: Comparing N=4 vs N=7
Given: Total system bandwidth S = 25 MHz = 200 channels (125 kHz/channel, GSM-like)
Cell radius R = 2 km
Compare N=4 and N=7 clusters
For N=4:
Reuse distance: D = 2 × √(3×4) = 2 × √12 = 2 × 3.46 = 6.93 km
Channels per cell: k = 200/4 = 50 channels/cell
D/R ratio: 6.93/2 = 3.46 (smaller → more co-channel interference)
For N=7:
Reuse distance: D = 2 × √(3×7) = 2 × √21 = 2 × 4.58 = 9.17 km
Channels per cell: k = 200/7 ≈ 28.6 ≈ 28 channels/cell
D/R ratio: 9.17/2 = 4.58 (larger → less co-channel interference)
N=4: 50 ch/cell, D/R=3.46 (more capacity, more interference)
N=7: 28 ch/cell, D/R=4.58 (less capacity, less interference)
Higher N → better quality but lower capacity per cell.

Cell Splitting — Increasing Capacity by Shrinking Cells

As urban areas become more congested, more capacity is needed without adding new spectrum. Cell splitting solves this by subdividing large cells into smaller cells:

Cell splitting: halving cell radius (R → R/2) quadruples the number of cells in the same area, quadrupling total capacity. Each cell still has its full channel allocation (k channels).
Cell Splitting Capacity Gain
Capacity ∝ 1/R²
→
Halving the radius (R → R/2) → area per cell decreases by factor 4 → 4× more cells in same geographic area → 4× capacity
→
Reducing radius by factor of k → capacity increases by k²
→
Cost: requires 4× as many base stations (capital expenditure)
→
Handoff rate increases (smaller cells → users cross cell boundaries more frequently)

Co-Channel Interference & the Q Ratio

The quality of cellular service is determined by the co-channel interference ratio (Q), also called the carrier-to-interference ratio when expressed in signal power terms:

Co-Channel Interference Ratio
Q = D/R = √(3N)
Q
Co-channel reuse ratio (dimensionless) — higher Q = more spatial separation before reuse = less interference
→
N=1: Q = √3 ≈ 1.73 (extreme interference — adjacent cells reuse same freq!)
→
N=3: Q = 3 (some separation)
→
N=7: Q = √21 ≈ 4.58 (widely used in analog/GSM cellular)
→
N=19: Q = √57 ≈ 7.55 (high quality, very few channels per cell)
Why CDMA Changed Everything

In FDMA/TDMA systems (1G/2G), different cells use different frequencies → N>1 required. In CDMA (3G CDMA2000, WCDMA), all cells use the same frequency but different spreading codes → N=1 frequency reuse is possible! Every cell can use the entire spectrum. The interference is managed through power control and code orthogonality, not geographic separation. This is a revolutionary increase in spectral efficiency.

Professor's Picky Corner — Frequency Reuse Math
  • The formula N = i² + ij + j² constrains valid cluster sizes. If you're given N=5, is it valid? Check: i=1,j=2: 1+2+4=7 (no). i=2,j=1: 4+2+1=7 (no). i=1,j=1: 1+1+1=3 (no). N=5 is NOT a valid cluster size. Valid values near 5: N=4 (i=2,j=0) and N=7 (i=2,j=1).
  • Cell splitting quadruples capacity but also quadruples handoff rate. If a user moves at 60 km/h and cells are R=500m radius, they cross a cell boundary every R/v = 500/16.7 = 30 seconds — a handoff every 30 seconds! The network signaling overhead becomes significant.
Given N=4, R=3 km, what is the minimum distance between two cells using the same frequency? D = R·√(3N) = 3·√12 = 3·3.464 = 10.39 km. This is the center-to-center distance. The edge-to-edge distance is D - 2R = 10.39 - 6 = 4.39 km. Watch out for the question asking edge-to-edge vs. center-to-center!
If an operator has 50 MHz of spectrum and uses N=7 with 200 kHz channels (like UMTS), how many channels per cell? Total channels = 50,000,000 / 200,000 = 250 channels. Channels per cell = 250/7 ≈ 35.7 → 35 channels/cell (floor, since you can't have fractional channels). Note: in CDMA, this math is different — all 250 codes are available in all cells simultaneously.

Mobile Phone & Base Station Functions

What happens inside each device — the radio functions both sides must perform

Mobile Phone (MS/UE) Functions

A mobile phone is far more than a microphone-speaker combination. The radio functions it must perform:

Radio Functions
  • Modulate/demodulate the RF carrier (convert bits ↔ radio waves)
  • Encode/decode — add FEC, detect/correct errors
  • Frequency synthesis — tune to the assigned channel frequency
  • Power amplification (TX) and low-noise amplification (RX)
  • Duplex operation — transmit and receive simultaneously (FDD) or alternately (TDD)
  • Timing synchronization — lock to BS timing reference
Network Functions
  • Cell selection — scan available channels, choose strongest BS
  • Registration — attach to network, report location to MSC
  • Authentication — prove identity via SIM-based challenge-response (A3/A8 algorithms in GSM)
  • Handoff/handover — decide when to switch BS (mobile-assisted)
  • Power control — respond to BS commands to adjust TX power
  • Location update — periodically report movement to network

Base Station Functions

The base station is the network's radio gateway. Its functions mirror and complement those of the mobile:

Radio Access

Manage multiple simultaneous radio connections using FDMA, TDMA, or CDMA. Transmit paging signals (broadcast channel) to alert idle phones of incoming calls. Broadcast system information (cell ID, frequency list, network ID).

Resource Management

Allocate channels to new calls. Manage traffic load — redirect to adjacent BS if overloaded (load balancing). Report capacity to BSC/RNC for resource management decisions.

Quality Control

Measure received signal quality (SINR, BER) from all connected mobiles. Send power control commands (increase/decrease TX power by 1 dB steps, up to thousands of times per second in CDMA). Trigger handoff measurements.

Professor's Picky Corner — MS/BS Functions
What is the difference between the Authentication Center (AuC) and the SIM card in GSM authentication? The SIM stores the subscriber's Ki (secret key, 128-bit). The AuC in the network also stores Ki for each subscriber. Authentication is done via a challenge-response (RAND + Ki → SRES using A3 algorithm). Neither the Ki nor SRES travels over the air — only the random challenge (RAND) and signed response (SRES). This prevents replay attacks.
Why do GSM phones broadcast their IMEI (device ID) during registration? IMEI (International Mobile Equipment Identity) is the hardware ID (not subscriber ID). It allows the network to block stolen phones via the EIR (Equipment Identity Register) — the network can blacklist an IMEI. But: IMEI is sent in plaintext in GSM — a security weakness. LTE/5G encrypt it earlier in the connection process.

The First Move: How a Mobile Finds & Connects

Step-by-step: power-on to first call — the complete cellular attach procedure

Startup Procedure: Finding a Network

When a mobile phone powers on, it has no idea which cell it is in, what frequency to use, or how to reach the network. Here is the complete startup sequence:

The phone scans all possible carrier frequencies in its supported bands. For each frequency where it detects a signal, it attempts to decode the Broadcast Control Channel (BCCH) — a logical channel on which every GSM base station continuously broadcasts system information.

From the BCCH, the phone learns: Cell ID (CGI), MCC (Mobile Country Code), MNC (Mobile Network Code), LAC (Location Area Code), available neighboring cells, and allowed access parameters.

The phone ranks cells by signal strength and "C1" path loss criterion. It selects the cell with the best C1 = received signal strength − minimum required signal strength − cell reselection offset. This is the "cell selection" algorithm.

Once a BS is selected, the phone must synchronize its timing with the base station. In GSM, it reads the Frequency Correction Channel (FCCH) and Synchronization Channel (SCH) to lock its internal clock to the BS's TDMA frame timing.

Timing advance: Because all mobiles in a cell are at different distances (different propagation delays), the BS must tell each mobile exactly how far in advance to transmit so all signals arrive at the BS at the correct TDMA slot. This timing advance value (0–63, representing 0–232 μs) is transmitted to the mobile.

The phone requests access via the Random Access Channel (RACH) — a shared contention channel. It sends a random access burst. The BS responds on the Access Grant Channel (AGCH) with a temporary channel assignment.

The phone sends a Location Update Request including its IMSI (or TMSI if it has one — TMSI is a temporary alias assigned by the network to protect IMSI privacy). The MSC authenticates the phone against the HLR/AuC. On success, the VLR records the subscriber's presence in this MSC area. The network sends a new TMSI to the phone.

When idle (no active call), the phone monitors the Paging Channel (PCH) periodically. The network knows which Location Area (group of cells) the phone is in from the most recent Location Update. When a call arrives for the phone, the MSC pages all BSs in the phone's Location Area. Each BS broadcasts the page (including the phone's IMSI or TMSI) on the PCH. The target phone responds via RACH, and the call setup proceeds.

Sleep mode: To save battery, phones don't continuously monitor the PCH. They use a Discontinuous Reception (DRX) cycle — wake up briefly to check the paging indicator, then sleep. The paging cycle is typically 1.28 seconds in GSM (the phone wakes for ~0.6 ms every 1.28 s).

When a call is initiated (mobile-originated) or received (mobile-terminated), the network assigns a dedicated traffic channel (TCH in GSM, DTCH in UMTS/LTE). The BS coordinates the precise frequency and time slot (FDMA+TDMA in GSM) or spreading code (CDMA) and power level.

Speech coding: Voice is compressed by a vocoder (coder-decoder). GSM uses Full Rate (FR) codec at 13 kbps, Half Rate (HR) at 5.6 kbps, Enhanced Full Rate (EFR) at 12.2 kbps, or Adaptive Multi-Rate (AMR) at 4.75–12.2 kbps. AMR adapts the bit rate based on channel quality and traffic load.

Professor's Picky Corner — Connection Procedure
Why does the phone transmit TMSI instead of IMSI in most messages? IMSI (International Mobile Subscriber Identity) is a permanent identity — if sent in plaintext over the air, it can be intercepted by IMSI catchers (fake base stations). The network assigns a Temporary Mobile Subscriber Identity (TMSI) and updates it regularly to prevent tracking. But TMSI is only valid in the current VLR area — when you roam, the IMSI must be sent once to the new VLR.
What is the Timing Advance (TA) used for, and what does TA=63 imply? TA compensates for propagation delay — farther phones must transmit earlier so their signals arrive at the BS aligned to the correct TDMA timeslot. TA is measured in units of 3.7 μs (one GSM bit period / 2). TA=63 → delay = 63 × 3.7 μs ≈ 233 μs → distance = 233 × 10⁻⁶ × 3×10⁸ / 2 ≈ 35 km. TA=63 is the maximum (defines maximum cell radius ≈ 35 km in GSM).

Handoff (Handover) — Complete Analysis

How active calls are transferred between base stations without interruption

What is Handoff?

Handoff (North American term) or Handover (European/international term) is the process of transferring an active call or data session from one base station to another as the mobile moves. It is the feature that distinguishes cellular from fixed wireless — without it, a call would drop every time you moved between cell boundaries.

Handoff execution: as MS moves from Cell A toward Cell B, BS-A's signal weakens and BS-B's strengthens. At the handoff point (crossing of signal levels, adjusted by hysteresis), the MSC transfers the call to BS-B.

Handoff Decision Criteria

Handoff is triggered by: (1) Received signal strength (RSS) from serving BS falling below threshold, (2) RSS from neighbor BS exceeding serving BS by more than hysteresis margin H (dB), (3) Distance-based (serving area boundary), (4) Traffic load (overflow handoff — load balancing). The hysteresis prevents "ping-pong" — repeatedly handing off back and forth as a user stands near a cell boundary.

Cellular Generations: 1G → 5G

The complete evolution — key technologies, speeds, and breakthroughs of each generation

The Five Generations — Overview Timeline

Cellular generation timeline: each generation defined by ITU/3GPP with specific minimum data rate, latency, and spectral efficiency targets

Generation-by-Generation Deep Dive

Technology: Analog FM radio. Frequency Division Multiple Access (FDMA) — each call uses its own dedicated frequency channel (30 kHz in AMPS).

Access Method: FDMA only. 832 channels in AMPS (416 duplex pairs). Each channel carries one voice call continuously (circuit-switched, analog).

Data: No data service. Analog modems over voice channels allowed very limited FAX/data (~2.4 kbps).

Security: None. Voice was transmitted as analog FM — anyone with a scanner could intercept calls. No authentication (SIM cards didn't exist) — phone cloning was trivial.

Limitations: Incompatible standards worldwide (AMPS in North America, NMT in Scandinavia, TACS in UK). No roaming between countries. Very limited spectrum efficiency. No encryption.

Key Revolution: Digital transmission. Voice is digitized (8 kHz × 8 bit/sample = 64 kbps PCM → compressed to 13 kbps by GSM FR codec). Digital enables encryption, error correction, and data services.

GSM Access: FDMA + TDMA. 200 kHz channels, 8 time slots per channel = 8 users per channel. Each user gets 1/8 of the time (full-rate) or 1/16 (half-rate).

IS-95 (cdmaOne): CDMA (Code Division Multiple Access). All users share the same 1.25 MHz band simultaneously, distinguished by orthogonal spreading codes. First commercial CDMA system.

Security: Authentication via challenge-response (A3 algorithm). Encryption (A5/1, A5/2 stream cipher — later broken but a major advance over 1G). User identity protection via TMSI.

Data: SMS (Short Message Service) — hugely popular. GPRS (2.5G): packet-switched data overlay on GSM, up to ~114 kbps. EDGE (2.75G): enhanced modulation (8-PSK vs GMSK), up to ~384 kbps.

Legacy: GSM became the dominant global standard — used by over 5 billion users at peak. Still used as fallback in many regions.

Key Revolution: Mobile internet. 3G was designed from the ground up for data services, not just voice. Defines IMT-2000 (ITU standard): 2 Mbps indoor, 384 kbps vehicular.

UMTS/WCDMA Access: Wideband CDMA. 5 MHz channels (vs 1.25 MHz in cdmaOne). Spreading factor variable (4–512). All users share entire 5 MHz simultaneously. Orthogonal Variable Spreading Factor (OVSF) codes assigned per user.

Key Features: Soft handoff (mobile connected to multiple BSs simultaneously). Power control at 1500 Hz — crucial for CDMA "near-far" problem management. QoS classes: conversational (voice), streaming (video), interactive (web), background (email).

HSPA (3.5G): High Speed Packet Access. HSDPA: downlink up to 14.4 Mbps using adaptive modulation (64-QAM), HARQ retransmission. HSUPA: uplink up to 5.76 Mbps. Introduced scheduling at 2ms intervals (turbo round-robin scheduling).

HSPA+ (3.75G): MIMO added, 64-QAM uplink, up to 42 Mbps downlink. Very competitive with early 4G LTE.

Key Revolution: All-IP network. 4G is purely packet-switched (no circuit-switched voice — voice over 4G is VoLTE, Voice over LTE, which is VoIP). All services (voice, data, video) are IP packets.

Access Method: OFDMA (Orthogonal Frequency Division Multiple Access) downlink — divides spectrum into many orthogonal subcarriers (15 kHz spacing), grouped into resource blocks (12 subcarriers × 7 symbols). Multiple users share these resource blocks dynamically per TTI (1 ms scheduling interval).

Uplink: SC-FDMA (Single Carrier FDMA) — like OFDMA but preserves lower peak-to-average power ratio (PAPR), important for battery-powered devices.

MIMO: Multiple-Input Multiple-Output antennas. 2×2 MIMO baseline, 4×4 in LTE-Advanced. Spatial multiplexing multiplies capacity (2×2 MIMO can double throughput). Beamforming directs signal to specific users.

Peak Rates: LTE: 100 Mbps DL, 50 Mbps UL. LTE-Advanced (IMT-Advanced 4G): 1 Gbps stationary, 100 Mbps mobile. Carrier aggregation: bond up to 5 carriers (5 × 20 MHz = 100 MHz bandwidth).

Flat Architecture: eNB (evolved Node B) connects directly to Evolved Packet Core (EPC). No BSC layer. Faster handoff via X2 interface (direct eNB-to-eNB). Latency < 10 ms (vs 50-100 ms in 3G).

Key Revolution: Three distinct use cases designed-in from the start: eMBB (enhanced Mobile BroadBand), URLLC (Ultra-Reliable Low-Latency Communications), mMTC (massive Machine-Type Communications).

eMBB: Peak 20 Gbps DL, 10 Gbps UL. Targets smartphones, AR/VR. Uses wide bandwidth (up to 400 MHz in FR2/mmWave), massive MIMO (up to 256 antennas), 64-QAM to 256-QAM.

URLLC: 1 ms round-trip latency, 99.9999% (6-nine) reliability. Targets autonomous vehicles, industrial automation, remote surgery. Achieved via mini-slots (as short as 1 OFDM symbol = 0.07 ms scheduling), HARQ with aggressive timelines, and network slicing.

mMTC: Supports 10⁶ devices/km². Targets IoT sensors. Uses NB-IoT (Narrowband IoT) and LTE-M as 5G-era IoT extensions. Very low power (10-year battery life target).

Spectrum: Two frequency ranges: FR1 (sub-6 GHz, 450 MHz–6 GHz) for coverage; FR2 (mmWave, 24–52 GHz) for capacity. New spectrum at 26 GHz, 28 GHz, 39 GHz.

Massive MIMO: 64–256 antenna elements at gNB. Enables 3D beamforming (elevation + azimuth steering), massive spatial multiplexing (serve many users simultaneously in same time/frequency), interference nulling.

Network Slicing: Virtualized networks-within-the-network. Operators can create isolated virtual networks with different QoS for different customers (e.g., one slice for emergency services, one for gaming, one for IoT).

Professor's Picky Corner — Cellular Generations
  • The slide notes that frequency reuse factor N=1 is used in 3G CDMA and 4G LTE. How is N=1 possible without massive interference? In CDMA (3G): spreading codes provide code-domain separation. If 1G used FDMA and 2G used FDMA+TDMA, why did efficiency increase? In 1G FDMA, each 30 kHz channel carries exactly 1 call (analog). IEEE 802.11 Wi-Fi Architecture

BSS, ESS, IBSS, AP, DS — the complete 802.11 network topology

802.11 Network Elements

The 802.11 standard defines a precise set of network components and their relationships. Understanding the exact terminology is critical — your professor will likely ask about specific element names:

Station (STA)

Any device with an 802.11 interface — laptop, phone, printer, IoT device. A STA is identified by its MAC address (48-bit, e.g., 00:11:22:33:44:55). STAs are the "users" of the wireless network.

Access Point (AP)

A special STA that provides wireless-to-wired bridging. The AP has two interfaces: (1) wireless radio interface (802.11), and (2) wired interface (usually Ethernet/802.3) connecting to the Distribution System. The AP manages the BSS and coordinates all STA transmissions.

Basic Service Set (BSS)

The fundamental building block of an 802.11 network. A BSS is a set of STAs that can communicate with each other. In infrastructure mode, the BSS consists of one AP + all STAs associated with it. Every BSS has a BSSID = the AP's MAC address (48-bit).

Distribution System (DS)

The backbone that connects multiple APs together. Usually an Ethernet LAN. The DS enables roaming — when a STA moves from one AP to another in the same ESS, the DS forwards frames to the new AP. The DS is NOT wireless in typical deployments (though 802.11s defines a wireless mesh DS).

Extended Service Set (ESS)

Multiple BSSs connected via a DS, appearing as one network to users. All BSSs in an ESS share the same SSID (network name, e.g., "OfficeWiFi") but may use different channels. The ESS has an ESSID = the SSID. STAs can roam between BSSs within the ESS. This is how enterprise Wi-Fi provides building-wide coverage.

Independent BSS (IBSS) / Ad-Hoc Mode

STAs communicate directly without an AP. The IBSS has no DS connection. STAs elect a temporary "IBSS master" for timing. Each IBSS has an IBSSID (randomly generated MAC-format address). Example: two laptops sharing files in a conference room with no Wi-Fi infrastructure available.

Professor's Picky Corner — 802.11 Architecture
What is the BSSID in infrastructure mode vs IBSS mode? Infrastructure mode: BSSID = AP's MAC address (assigned by manufacturer, globally unique 48-bit address). IBSS (ad-hoc) mode: IBSSID = locally administered MAC address (randomly generated, with the LA bit set). Crucially: in IBSS mode, the BSSID does NOT match any device's MAC — it's a synthetic identifier for the ad-hoc network.
Can a STA belong to two BSSs simultaneously? Generally no — a STA is associated with exactly one AP at a time in standard operation. However: (1) multi-band devices can have separate associations on 2.4 GHz and 5 GHz simultaneously if they're treated as separate radios, and (2) 802.11v/k/r protocols allow pre-association scanning in preparation for roaming, but not simultaneous dual association.
What is the difference between SSID, BSSID, and ESSID? SSID (Service Set ID) = human-readable network name ("HomeNetwork", up to 32 characters). BSSID = MAC address of the AP (machine-readable, 48-bit). ESSID = same format as SSID but refers to the extended service set name (shared across all APs in the ESS). In practice, SSID and ESSID are often used interchangeably, but technically ESSID is the identifier for the entire multi-AP network.

Channel Structure — 2.4 GHz (Critical for Exam)

The 2.4 GHz band (2.400 – 2.4835 GHz = 83.5 MHz total) is divided into 14 channels (13 in Europe, 11 in North America). Each channel is 22 MHz wide but spaced only 5 MHz apart — meaning adjacent channels overlap significantly:

2.4 GHz Wi-Fi channels: 11 channels in North America, 13 in Europe. Only Ch 1, 6, and 11 are non-overlapping (25 MHz separation needed for 22 MHz channels). All other adjacent channels interfere.
Why 802.11a/ac at 5 GHz has More Non-Overlapping Channels

The 5 GHz U-NII bands span ~580 MHz (in the US/EU). 802.11a/n/ac uses 20 MHz channels (non-overlapping), 40 MHz (in 802.11n), 80 MHz (802.11ac), or 160 MHz (802.11ac wave 2). In the 5 GHz band: 20 MHz channels → up to 25 non-overlapping channels (varies by region/country). This is why 5 GHz has far less congestion than 2.4 GHz — more channels to distribute the many networks in a dense apartment building.

OFDM — The Key Technology Behind 802.11a/g/n/ac/ax

Starting with 802.11a, Wi-Fi uses OFDM (Orthogonal Frequency Division Multiplexing). Understanding why OFDM replaced DSSS is critical:

Problem: ISI from Multipath

Indoor multipath delay spread is 50-100 ns. At 11 Mbps (802.11b DSSS), symbol period = 91 ns — dangerously close to the delay spread. At 54 Mbps, symbol period = 18.5 ns — far shorter than delay spread → severe ISI.

Solution: OFDM

OFDM divides the 20 MHz channel into 52 subcarriers (48 data + 4 pilot), each 312.5 kHz wide. Each subcarrier's symbol period = 1/312.5kHz = 3.2 μs — 32× longer than delay spread (100 ns). ISI is negligible. A cyclic prefix (0.8 μs guard interval) absorbs the remaining multipath echoes.

OFDM Subcarrier Count — 802.11 Specifics

802.11a/g: 64 total subcarriers (52 active: 48 data + 4 pilot + 12 null/guard). 802.11n: Up to 108 subcarriers in 40 MHz mode. 802.11ac: 256 subcarriers (234 data) in 80 MHz; 512 (468 data) in 160 MHz. 802.11ax: Same subcarrier spacing but adds OFDMA — the subcarriers can be allocated to different users simultaneously (like LTE Resource Blocks).

Professor's Picky Corner — 802.11 Variants
  • 802.11b and 802.11g are backward compatible (g devices can talk to b devices at lower rates). 802.11a is NOT compatible with 802.11b/g (different band: 5 GHz vs 2.4 GHz). 802.11n is compatible with both b/g (at 2.4 GHz) and a (at 5 GHz). This compatibility question is a classic trap.
  • Why was 802.11a released at the same time as 802.11b (1999) but never as popular? 802.11a used 5 GHz which had less range than 2.4 GHz (higher frequency = more absorption through walls) and chipsets were initially more expensive. 802.11b at 2.4 GHz penetrated walls better and was cheaper → dominated. By the time 802.11a's advantages (more channels, less interference) were appreciated, 802.11g provided equivalent speeds at 2.4 GHz.
Why must non-overlapping Wi-Fi channels be at least 5 channels apart (e.g., 1 and 6 separated by 5)? Each channel is 22 MHz wide but centered at intervals of 5 MHz. Channel 1 center = 2.412 GHz, Channel 6 center = 2.437 GHz. Separation = 25 MHz. For 22 MHz channels to be non-overlapping, the gap between their edges must be ≥ 0: 25 MHz - 22 MHz = 3 MHz of guard band. This is barely sufficient. Channels 1, 6, 11 in North America meet this criterion. Using 1 and 5 (20 MHz apart) means partial overlap (22 MHz channel extends ±11 MHz from center: Ch1 spans 2.401-2.423, Ch5 spans 2.421-2.443 → 2 MHz overlap).

IEEE 802.11 MAC: CSMA/CA Deep Dive

How Wi-Fi shares the wireless medium — DCF, backoff, RTS/CTS, and frame acknowledgment

Why CSMA/CA? — Revisited in 802.11 Context

802.11 uses CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance) as the default medium access control. The primary MAC protocol is the DCF (Distributed Coordination Function).

Key insight: In 802.11, unlike wired Ethernet, collisions cannot be detected during transmission (the transmitter's own signal overpowers any collision). Therefore, instead of detecting and recovering from collisions, 802.11 actively tries to avoid them through probabilistic backoff, and uses explicit ACKs to confirm successful delivery.

DCF Operation — Step by Step

DCF timing: STA A wins contention (smaller backoff) and transmits. STA B detects channel busy, freezes its backoff counter. After A's ACK, B waits DIFS and resumes remaining backoff before transmitting.

RTS/CTS: Solving the Hidden Terminal Problem

RTS/CTS mechanism: A sends RTS to AP. AP broadcasts CTS to all. Hidden node C hears CTS and sets its NAV (Network Allocation Vector) timer — it will not transmit until NAV expires. Collision at AP is avoided even though C couldn't hear A's RTS.
NAV — Network Allocation Vector

The NAV is a virtual carrier sense mechanism. When a STA hears a CTS (or any frame with a Duration field), it sets a timer equal to the Duration field value. During this time, the STA will not transmit — even if it senses the channel as idle (because the actual transmission it would interfere with may be out of range). NAV enables "virtual carrier sense" for nodes that can't physically sense the ongoing transmission but know one is happening.

RTS/CTS Overhead — When to Use It:

RTS/CTS adds overhead: each DATA frame now requires RTS + CTS + DATA + ACK = 4 frames. For small frames, this overhead can exceed the benefit. 802.11 implements an RTS threshold — RTS/CTS is only used when the data frame exceeds this threshold (default typically 2347 bytes → disabled by default). For small IoT packets (100 bytes), RTS/CTS overhead is ~30× the payload. For large video frames (1500 bytes), the overhead is small relative to the protection benefit.

Professor's Picky Corner — CSMA/CA & DCF
  • SIFS < DIFS. This priority ordering means: ACK, CTS, and BlockACK (high priority) use SIFS (respond immediately after SIFS). Normal data uses DIFS (waits longer → lower priority). 802.11e/WMM adds AIFS (Arbitration IFS) for QoS: voice uses AIFS=SIFS+2×slot < DIFS, while background uses AIFS=SIFS+7×slot> DIFS. So voice frames win channel access statistically.
  • Binary exponential backoff is not "fair" in the short term — whoever draws the smallest backoff wins. It is fair in the long term — over many transmissions, each STA wins roughly equally often (assuming equal CW sizes).
If two STAs draw the same backoff value and transmit simultaneously, a collision occurs. Neither transmitter detects this (no CD). How does 802.11 handle this? The transmitter sets a timer for ACK_Timeout (default ~340 µs in 802.11a). If no ACK is received within this time, the transmitter infers a collision or channel error occurred. It then enters the retry procedure: doubles CW, draws new random backoff, and retransmits. After RetryLimit retransmissions without success (7 short + 4 long), the frame is dropped and the upper layer is notified.
Does CSMA/CA guarantee no collisions? No! The backoff reduces collision probability but does not eliminate it. Two STAs can independently draw the same backoff value and transmit simultaneously, causing a collision. The protocol detects this via missing ACK and recovers via retransmission with larger CW. CSMA/CA is probabilistic collision avoidance, not deterministic collision prevention.

Scanning & Association in 802.11

How a Wi-Fi device finds and connects to a network — Passive vs Active scanning

The Two Scanning Methods

Before a STA can associate with an AP, it must discover available networks. 802.11 defines two scanning modes:

Passive Scanning

How: STA tunes to each channel and listens for Beacon frames. APs transmit Beacons every 100 ms (default TU = 1024 µs ≈ 1.024 ms, so every ~97 TU). Beacon contains: SSID, BSSID, supported rates, channel, security capabilities, timestamp, beacon interval.

Pros: STA transmits nothing — saves power. Cannot be detected by AP (passive).

Cons: Slow — must listen on each channel for at least one full beacon interval (100 ms per channel). For 11 channels at 2.4 GHz = ~1.1 seconds total scan time.

Active Scanning

How: STA broadcasts a Probe Request frame on each channel (can include a specific SSID to find, or wildcard SSID to find all). Any AP on that channel with matching SSID replies with a Probe Response (same content as Beacon).

Pros: Fast — doesn't need to wait for beacon interval. Response arrives within PIFS + propagation ≈ ~300 µs per channel.

Cons: STA transmits (uses power). SSID in Probe Request is visible — privacy concern (reveals which networks the device has previously connected to).

Privacy Issue with Active Scanning

When a device sends a Probe Request with a specific SSID (e.g., "Starbucks_WiFi"), it reveals which networks the device has previously connected to. With many APs logging Probe Requests, it's possible to track users' movement patterns and history. Modern devices (iOS 14+, Android 10+) use random MAC addresses in Probe Requests ("MAC randomization") and often send broadcast (empty SSID) Probe Requests to mitigate this tracking.

The Full Association Procedure

After scanning, a STA must go through three states to become fully connected:

802.11 defines two authentication methods: Open System Authentication (essentially no authentication — any STA can authenticate) and Shared Key Authentication (WEP-based challenge-response — now deprecated, broken).

Modern security (WPA/WPA2/WPA3) is NOT implemented at this state — it's implemented at the association/4-way handshake stage. The 802.11 authentication state machine is a vestigial mechanism; real security happens during the RSNA (Robust Security Network Association) procedures after association.

STA sends an Association Request to the chosen AP. Request includes: SSID, supported rates, RSN IE (Robust Security Network Information Element — security capabilities), HT/VHT/HE capabilities (MIMO capabilities), power save mode.

AP responds with Association Response: status code (0 = success), Association ID (AID — 14-bit identifier used for Power Save polling), supported rates intersection.

After successful association, the AP maps the STA's MAC address to its port in the DS. From this point, the AP can forward frames from DS to STA and vice versa.

After association, WPA2/WPA3 performs the 4-Way Handshake to derive session keys (PTK — Pairwise Transient Key) from the PMK (Pairwise Master Key).

PMK derivation: WPA2-Personal: PMK = PBKDF2(SSID, passphrase, 4096 iterations) — this is why short passphrases are weak (dictionary attacks). WPA2-Enterprise: PMK derived from 802.1X/EAP authentication (RADIUS server validates user credentials).

4-Way Handshake: AP sends ANonce → STA computes PTK, sends SNonce + MIC → AP computes PTK, sends GTK (Group Temporal Key for broadcasts) + MIC → STA installs keys, sends ACK. After this, all data frames are encrypted with AES-CCMP (WPA2) or AES-GCMP (WPA3).

Professor's Picky Corner — Scanning & Association
What is a hidden SSID (network with SSID broadcast disabled)? In passive scanning, the AP sends Beacons with an empty SSID field (length=0). The network is "hidden." However: (1) The BSSID is still visible in Beacon frames, (2) When a STA connects, the SSID is visible in the Probe Request and Association Request in plaintext. Hidden SSIDs provide minimal security benefit (they're easy to discover by watching existing client traffic) but reduce network visibility in Wi-Fi scanners. NOT recommended as a security measure.
During roaming within an ESS, does a STA need to re-authenticate and re-associate each time it switches APs? In basic 802.11, yes — the STA must send a Reassociation Request (not Association Request) to the new AP, which triggers the 4-Way Handshake again. 802.11r (Fast BSS Transition) accelerates this by pre-caching keys and allowing the handshake to happen before the STA actually moves to the new AP. 802.11r reduces handoff delay from ~100-200 ms to ~20-30 ms.

Bluetooth (IEEE 802.15.1) — Complete Analysis

The piconet architecture, FHSS operation, and comparison with Wi-Fi

Bluetooth Architecture: Piconet & Scatternet

Bluetooth creates ad-hoc networks called piconets. Unlike Wi-Fi, there is no central AP — but Bluetooth is not fully symmetric either. One device takes the role of master and up to 7 active slaves plus 255 parked slaves can participate:

Master Device

Sets the hopping frequency sequence and timing. All slaves synchronize to the master's clock (Bluetooth uses a 28-bit clock with 312.5 µs ticks = 1600 hops/second). The master controls which slave can transmit in each slot (TDD-based polling).

In a piconet, communication is ONLY master↔slave. Slave-to-slave communication goes through the master. The master's device address (BD_ADDR) seeds the FHSS hopping sequence.

Scatternet

A device can participate in multiple piconets simultaneously — it would be a master in one and a slave in another. The overlapping piconets form a scatternet. This enables multi-hop ad-hoc routing without a fixed infrastructure.

Scatternets are theoretically defined by the standard but rarely implemented in practice — timing coordination between piconets is complex and the throughput penalty is significant.

Frequency Hopping Spread Spectrum (FHSS)

Bluetooth's signature technology is FHSS (Frequency Hopping Spread Spectrum). The radio hops among 79 channels (each 1 MHz wide) in the 2.4 GHz ISM band at 1600 hops/second:

FHSS Parameters
→
79 channels × 1 MHz = 79 MHz total hopping bandwidth (2.402 – 2.480 GHz)
→
Hop rate: 1600 hops/second → dwell time per channel = 625 µs (one time slot)
→
Hopping sequence: pseudo-random, seeded by master's BD_ADDR + clock → only piconet members can predict next frequency
→
AFH (Adaptive Frequency Hopping, BT 1.2+): detects occupied channels (occupied by Wi-Fi) and excludes them from the hop sequence. BT and Wi-Fi coexist much better with AFH.
Why FHSS Instead of DSSS or OFDM?

FHSS provides: (1) Interference resistance — if a hop lands on a busy channel, only that 625 µs slot is lost; the next hop is likely clear. (2) Security — without knowing the hopping sequence (seeded by master's address), an outsider cannot easily listen to or jam the connection. (3) Simplicity — the early Bluetooth chipsets (1999) were simpler to implement with FHSS than with OFDM. Modern Bluetooth (BR/EDR) still uses FHSS; Bluetooth LE uses GFSK on fixed channels (2, 26, 80 for advertising; 37 hop channels for data).

Professor's Picky Corner — Bluetooth
Why can a Bluetooth piconet have only 7 active slaves but up to 255 parked slaves? The active member address (AM_ADDR) field is 3 bits wide, so it can distinguish 2³ = 8 values, one of which is reserved for broadcast — leaving 7 unique active-slave addresses. Parked devices use a wider 8-bit field (PM_ADDR), giving 2⁸ − 1 = 255 addressable parked slots. A parked device is still clock-synchronized to the master and can be "woken" into an active slot, but does not occupy one of the 7 active addresses while parked. Exam trap: don't confuse "7 slaves" (a hard architectural ceiling per piconet) with "100,000 nodes" (a property of Wireless Sensor Networks, an entirely different single-hop vs. multi-hop category).
If Bluetooth and Wi-Fi both use the unlicensed 2.4 GHz ISM band, why don't they destroy each other's signal? Three reasons working together: (1) Bluetooth's FHSS spreads its energy thinly across 79 × 1 MHz channels rather than sitting on one wide channel, so any single collision only costs one 625 µs slot. (2) Adaptive Frequency Hopping (AFH), introduced in Bluetooth 1.2, actively senses which channels Wi-Fi is using and removes them from Bluetooth's hop set. (3) Wi-Fi's CSMA/CA (covered in the 802.11 section) makes Wi-Fi itself defer when it senses the channel busy — so the burden of avoidance is shared by both sides, not just Bluetooth. A picky professor may ask you to name all three, not just one.
Bluetooth Classic vs. BLE — why does BLE only support 2 Mbps (a max LOWER than Classic's 3 Mbps EDR) yet is marketed as the modern standard? BLE was never designed to maximize throughput; it was designed to minimize energy per bit and connection-setup latency. A coin-cell battery-powered heart-rate sensor doesn't need 3 Mbps — it needs to sleep almost all the time and wake for a few milliseconds to send a tiny payload. The relevant comparison metric for BLE is µW of average power, not Mbps of peak rate. This is a classic professor trap: he may ask "which technology has the higher data rate" hoping you say BLE because it sounds newer — the correct answer is Classic/EDR.

Wireless Mesh Networks (WMN)

Multi-hop, infrastructure-based — fixed routers relay traffic so coverage can grow without every node reaching the gateway directly

Where This Fits in the Taxonomy

Recall the 2×2 grid from the very first sections: single-hop vs. multi-hop crossed with infrastructure-based vs. infrastructure-less. Cellular, Wi-Fi, and WiMAX were all single-hop infrastructure-based — one radio jump straight to a base station. Wireless Mesh Networks are the first multi-hop infrastructure-based system in the course: there is still a backbone of fixed "base-station-like" devices, but a message may now bounce through several of them before reaching the wired network.

Network Structure — Two Device Roles

A WMN is built from exactly two kinds of nodes, and the professor loves asking you to identify which is which from a scenario description:

Mesh Routers (MRs)

Fixed (non-mobile) nodes that form the backbone of the mesh. They are often the only nodes with a wired connection to the internet — these act as gateways. Mesh routers forward traffic for other routers and for clients, even traffic that isn't addressed to them.

Mesh Clients (MCs)

End-user devices — laptops, phones, sensors. They connect to a nearby mesh router (like connecting to a Wi-Fi AP) or directly to another client. Clients are typically mobile and do not need to be powered continuously.

The Three Properties That Define a Mesh

Multi-hop Routing

Data travels hop-by-hop through multiple mesh routers. If no direct path exists to the destination, intermediate nodes forward the packet — exactly like routers on the wired internet, except every "router" here is also a radio.

Self-Healing

If a node fails or a link breaks, the network automatically reroutes traffic through alternative paths. This is only possible because multiple routers exist between any two points — remove the single point of failure that a one-hop cellular/Wi-Fi base station represents.

Self-Configuration

New nodes can join and immediately start routing without manual setup. Each node participates in neighbor discovery and routing — there is no central controller deciding the topology in advance.

A single gateway router (MR-GW) provides the only wired link to the internet. Every other router relays traffic multiple hops to reach it — exactly the "multi-hop infrastructure-based" structure that separates a mesh from single-hop Wi-Fi/cellular.

The Hop-Count Bandwidth Tradeoff — Why "Limited Bandwidth" Deserves Special Attention

Because every mesh router shares the same radio medium for both receiving a packet from one hop and relaying it to the next, a single end-to-end flow that travels N hops consumes roughly N times the channel airtime of a single-hop transmission. A common exam-style estimate: if a single-hop link can sustain a raw rate R, an N-hop path (with a single shared channel and no separate radios per hop) achieves an effective end-to-end throughput on the order of R / N. This is precisely why mesh networks favor multi-radio, multi-channel backbones (802.11s, WiMAX, LTE) — giving each hop a different channel avoids this self-interference penalty.

Real-World Application Patterns

The slides show six application scenarios for WMNs back-to-back — military, maritime, hybrid community broadband backbone, drone (UAV) mesh, large-scale agricultural monitoring with mobile machinery, and smart/connected cities.   • At least one router has a wired/backhaul connection

  • No node is pre-designated as a router — roles emerge dynamically

Professor's Picky Corner — Wireless Mesh Networks
A scenario says: "Sensors are placed in a field; some are fixed solar-powered relay towers, others are mobile tractors with sensors that connect to the nearest tower; the towers forward all data toward one tower with cellular backhaul." Is this a Mesh Network or a Wireless Sensor Network? It's a textbook Wireless Mesh Network, NOT a generic WSN, because there are two clearly distinct roles (fixed relay towers = mesh routers, mobile tractor sensors = mesh clients) and one designated gateway. A true WSN (multi-hop infrastructure-LESS) would have sensor nodes themselves relaying for each other with no distinct "router" class and typically no guaranteed gateway — exactly the agricultural monitoring example from the slides, which is explicitly listed as a WMN application precisely because of the fixed-router structure.
Why is "self-healing" listed as a feature of mesh networks but not of single-hop cellular networks? In cellular, if your serving base station fails, you do not get re-routed through another mobile's phone — you simply lose service until you connect to a different base station via a fresh single-hop link. There is no concept of "rerouting around a failure" within the wireless hop itself because there's only one hop. Self-healing requires multiple wireless hops between the source and the backbone — a property unique to multi-hop topologies (mesh and ad-hoc), not single-hop ones.

WiMAX (IEEE 802.16)

Single-hop, infrastructure-based — the "last-mile" wireless broadband alternative to cable and DSL

Where This Fits in the Taxonomy

WiMAX belongs in the same taxonomy box as Cellular Networks and Wi-Fi: single-hop, infrastructure-based. A subscriber device talks directly to one base station, which is wired into the larger network — there is no relaying through other subscriber devices. What makes WiMAX distinct from Wi-Fi is range: WiMAX base stations are designed to cover kilometers, not just a building, making it a metro-area technology rather than a local-area one.

What WiMAX Actually Is

WiMAX stands for Worldwide Interoperability for Microwave Access. It is not itself a single protocol — it is a certification mark/brand for interoperable equipment that implements the IEEE 802.16 family of standards, the same relationship that "Wi-Fi" has to IEEE 802.11.

Headline Numbers
→
Original design target: 30–40 Mbps per channel
→
2011 update (802.16m, marketed as "WiMAX 2"): up to 1 Gbit/s for fixed stations
→
Underlying standards family: IEEE 802.16
A WiMAX base station can reach subscribers either by direct line-of-sight microwave (high throughput, requires clear path) or by relaying through non-line-of-sight paths at lower throughput — the same fundamental tradeoff every microwave/RF system faces.

What WiMAX Is Used For

The slides list four concrete use-cases — group them as "what problem does WiMAX solve that Wi-Fi and cellular don't":

Last-Mile Broadband

A wireless alternative to cable and DSL for the "last mile" connecting a home/business to the ISP — useful where trenching fiber or copper is expensive or impossible (rural areas, developing regions, disaster-damaged infrastructure).

Metro Mobile Broadband

Portable mobile broadband connectivity across cities and even countries — positioning WiMAX (especially mobile WiMAX, 802.16e) as a competitor to early 4G/LTE deployments.

Triple-Play Services

Data, telecommunications (VoIP), and IPTV over the same wireless link — "triple play" because the operator bundles internet + phone + TV over one infrastructure.

Business Continuity

A backup source of internet connectivity if a business's primary wired link fails — since WiMAX needs no physical cable run, it can be activated quickly as disaster-recovery connectivity.

Professor's Picky Corner — WiMAX
Is WiMAX the same thing as 4G/LTE? No — this is a very common confusion. WiMAX (802.16) and LTE are competing, parallel standards that both emerged to deliver mobile/metro broadband around the same era. They are not the same technology family: LTE descends from the 3GPP cellular standards lineage (GSM → UMTS → LTE), while WiMAX descends from the IEEE 802 standards lineage (the same family tree as Wi-Fi). In the end, LTE won the global market and WiMAX deployments became comparatively rare, but on an exam you must NOT call WiMAX "a type of 4G" — it is a separate, competing single-hop infrastructure-based technology that happened to target a similar market.

All three have "dynamic topology" but for different physical reasons — this is the detail a picky exam loves. Wireless Sensor Networks (WSN) — Deep Dive

WSNs sit under the umbrella of Low-Power Wireless Personal Area Networks (Low-PAN), standardized under IEEE 802.15 (the same numbering family as Bluetooth, which is 802.15.1). The single sentence that captures the entire design philosophy:

The Core Design Constraint

Energy is the primary constraint — every other design decision in a WSN traces back to squeezing out every microjoule, because nodes run on small, often non-rechargeable batteries that may need to last months or years unattended.

Structural Properties
  • Nodes powered by non-rechargeable batteries
  • Data flows usually to one centralized location (a "sink")
  • Low per-node data rate, but networks can scale to up to 100,000 nodes
  • Data is highly correlated in time and space — neighboring sensors often sense similar things
  • Nodes can cooperate in transmission, reception, compression, and signal processing (in-network processing)
Design Consequences
  • Short-range links must budget transmit, circuit, AND processing energy — not just radio power
  • Sophisticated signal-processing techniques are not necessarily energy-efficient, even if they're bandwidth-efficient
  • Sleep modes save energy but complicate networking (a sleeping node can't relay for its neighbors)
  • Fundamental tradeoffs: delay vs. throughput vs. node/network lifetime
  • Requires careful optimization of node cooperation strategies

Some planning exists but the exact layout adapts to terrain or mission needs. Example from slides: Precision Agriculture is also tagged semi-structured in places — the row/grid plan is "structured" in intent but must adapt to field topology.

Unstructured

No predetermined placement — nodes are scattered/dropped and must self-organize entirely on their own. Example from slides: Military battlefield tactical monitoring (sensors remotely deployed to monitor enemy troop movements, with no opportunity to plan exact placement).

Delay Tolerant Networks (DTN)

Multi-hop, infrastructure-less — designed for networks where an end-to-end path may never exist at any single instant

The Core Idea in One Sentence

Ordinary internet routing assumes you can find some path from source to destination right now. DTN abandons that assumption entirely: it assumes no contemporaneous end-to-end path may exist, and instead lets intermediate nodes store data for an extended period until a forwarding opportunity arises — even if that opportunity is hours or days away.

Store-and-Forward Message Switching

This is the single mechanism that makes DTN possible, and it is named explicitly and illustrated with a figure on the slides — treat it as guaranteed exam material:

The DTN Forwarding Rule
1
Whenever forwarding IS currently possible, each received data packet ("bundle") is forwarded immediately
2
If forwarding is NOT currently possible, but is expected to become possible in the future, the bundle is stored locally for future transmission
3
DTN architecture expects nodes to be able to store bundles for an extended period — this requires meaningfully more persistent storage than a typical internet router buffer (which holds packets for milliseconds, not hours/days)
Each hop forwards when a link is available and buffers when it isn't. The end-to-end "path" only exists across time, never at one fixed instant.

DTN vs. Conventional Internet Architecture

The slides include a direct side-by-side that is extremely exam-friendly because it's compact and structured around clear opposites:

Property Internet Environment DTN Environment
End-to-end RTT Not large Very large delays
Path Availability Some path exists between endpoints (at any given time) Intermittent and scheduled links only
Reliability Mechanism End-to-end reliability via ARQ works well No ARQ — different network architectures needed
Switching Abstraction Packet-switching is the right abstraction Conversational protocols fail entirely
Why ARQ Fails and "Conversational" Protocols Fail

ARQ (Automatic Repeat reQuest) assumes a sender can retransmit quickly after detecting loss — it needs a feedback loop that returns within a reasonable time. If the round-trip time is hours (e.g. a deep-space link, or a sensor that only connects once a day), ARQ's retransmission timers become meaningless. Likewise, "conversational" protocols like TCP's three-way handshake assume both ends are reachable closely together in time to exchange SYN/SYN-ACK/ACK — if the destination isn't even reachable yet, there is no conversation to have. DTN replaces this with bundles that are self-contained enough to be stored and relayed without a live back-and-forth.

DTN Challenges

Application Assumptions Break

Applications assume end-to-end "connections" exist — DTN breaks this assumption at the architecture level, so applications must be redesigned around bundles, not sockets.

Routing Doesn't Understand Schedules

Conventional routing protocols don't natively reason about "this link will be available at time T" — DTN routing must incorporate scheduling knowledge to route bundles toward links that will exist in the future.

Long Disconnection & Poor Performance

Long-term disconnection causes failures in naive designs; end-to-end reliability is poor with high loss rates, and end-to-end performance is poor with high delays — all downstream consequences of abandoning the "always-connected" internet assumption.

DTN Applications

Rural / Developing-Region Connectivity

Mobile-enabled delay-tolerant networking provides internet-like connectivity in rural developing regions where continuous infrastructure doesn't exist — vehicles or people physically carry data between disconnected pockets of connectivity.

ZebraNet — A Real Deployment

A tracking node (CPU, flash storage, radio) is attached to zebras to record position/data and uses store-and-forward communication to relay data when nodes come into range of each other or a base station. First deployed in 2004 in Kenya. This is the deck's named real-world case study — expect the professor to ask "name the real DTN deployment example from the slides."

Satellite Systems as a DTN-Adjacent Example

The slides also place satellite systems in the DTN applications discussion: satellites cover very large areas at different orbit heights — GEO (~39,000 km) versus LEO (~2,000 km) — and are often optimized for one-way transmission such as radio (XM, DAB) and movie/TV broadcasting (SatTV). The connection to DTN is conceptual: very large delays and scheduled/intermittent visibility windows (a satellite is only "in view" for part of its orbit) are exactly the conditions DTN is designed around.

A professor asks: "Why can't you just use TCP/IP over a link to Mars?" The expected answer chain: (1) round-trip light-time delay to Mars is many minutes, far exceeding TCP's retransmission timeout assumptions; (2) TCP's three-way handshake is a "conversational" protocol requiring near-real-time back-and-forth, which breaks down when the other side may not even be reachable at handshake time; (3) the path itself may not exist continuously — planetary rotation and orbital geometry mean a ground station and a relay satellite are only mutually visible during certain windows. The correct framing is "TCP/IP assumes a contemporaneous end-to-end path with manageable RTT; deep space violates both assumptions, which is exactly the DTN architecture problem."

Where This Fits in the Taxonomy

VANET (Vehicular Ad-hoc Network) is explicitly defined on the slides as a form of Mobile Ad-hoc Network (MANET) — so it inherits the multi-hop, infrastructure-less classification, though in practice many real VANET deployments blend in fixed Road-Side Units (RSUs) for partial infrastructure support. This is the most heavily illustrated topic in the entire deck (over 20 slides), so expect proportionally heavy exam coverage.

What VANET Provides

VANET provides communication: (1) among nearby vehicles, and (2) between vehicles and nearby fixed equipment. This two-part definition is the seed for the entire V2V/V2I split covered below.

Road-Side Units (RSUs) — The Fixed Half of the Picture

Road-Side Infrastructure Units (RSUs) are fixed equipment placed along roads, equipped with on-board processing and wireless communication modules. They serve as the "base station" half of vehicular communication — the part that introduces infrastructure into an otherwise infrastructure-less ad-hoc model.

Vehicle Platooning — The Deck's Second Featured Example

Platooning gets its own multi-slide deep dive comparing three braking technologies, because it's a clean illustration of WHY V2V communication beats sensor-only approaches:

Approach How Braking Information Travels Limitation
Manual Driving Human perception & reaction only Significant brake lag, requires larger following distances
Radar-Based Systems On-board sensors detect the car ahead decelerating Reduces reaction time but still has processing/detection delay
V2V-Based Platooning Braking info shared wirelessly & instantly across all vehicles Enables near-simultaneous braking, minimal lag, closer spacing
Why This Matters: Speed-of-Information vs. Speed-of-Physical-Propagation

Radar can only tell a following car that the car immediately ahead is decelerating — that information must then ripple backward car-by-car at the speed of each car's own sensor-and-reaction loop. V2V broadcasts the braking event nearly instantaneously to every vehicle in radio range simultaneously, collapsing the "ripple delay" that otherwise accumulates down a long line of cars. This is the core mechanism behind two platooning benefits explicitly named on the slides: faster, synchronized braking response (enhancing safety, enabling closer following distances, improving traffic flow) and improved fuel efficiency through reduced aerodynamic drag (closer spacing at highway speed reduces wind resistance for trailing vehicles, similar to drafting in cycling/racing).

Radar's chain reaction adds up reaction delay car-by-car down the line; V2V's broadcast nature delivers the braking event to every vehicle in range at once, collapsing that cumulative delay.

Professor's Picky Corner — VANET / C2X / ITS
Is VANET the same as a Wireless Mesh Network, since RSUs sound like fixed mesh routers? No, and this is one of the most exploitable confusions on the exam. VANET is officially classified as multi-hop infrastructure-LESS (a form of MANET) precisely because the core V2V capability does NOT depend on RSUs existing — two vehicles can communicate directly with no fixed equipment involved at all. RSUs are an OPTIONAL infrastructure enhancement layered on top (the V2I half), not a structural requirement the way mesh routers are required for a Wireless Mesh Network to function. If a question describes a pure V2V scenario with zero RSUs, it is still VANET, but it would NOT qualify as a Mesh Network under this course's definition, because there is no dedicated router/client role split.
A diagram shows a car braking, with a wireless signal radiating outward in all directions toward nearby cars simultaneously rather than just to the car directly behind. What capability is this testing? This tests recognition of broadcast (not unicast) V2V messaging. Safety-critical platooning/cooperative-perception messages are broadcast to all vehicles in range rather than addressed point-to-point, because the sender often doesn't know in advance exactly which specific following vehicles need the warning — broadcasting lets every receiver in range decide locally whether the information is relevant to them. This is structurally different from the radar chain-reaction model, where information only reaches the IMMEDIATE next vehicle and must rebroadcast/re-detect at each hop.
Why does the Challenges slide list "unidirectional links" under Communication, when most of the chapter discusses two-way V2V communication? This is a subtle trap: not every V2X link is symmetric. A fast-moving vehicle passing a fixed RSU, or two vehicles passing each other at high relative speed in opposite lanes, may have a connection window so brief or asymmetric (e.g. different transmit power, different antenna orientation, different interference at each end) that the link only reliably carries data in one direction at a given moment, even though the underlying radios are technically capable of both directions. This matters for protocol design: you cannot always assume a reliable two-way handshake will complete before the link disappears, which again connects back to the DTN section's "no contemporaneous end-to-end path" theme — another sign the exam likes connecting topics across the whole deck.

Cross-Topic Threads the Exam Likes to Pull

A "comprehensive" exam built from a deck this dense will often ask you to connect ideas across sections rather than recall one section in isolation. These are the threads that recur most:

The Hop-Count Tradeoff

Appears in cellular frequency reuse (capacity vs. interference), Wi-Fi channel access (contention grows with users), and mesh networks (bandwidth shrinks ~1/N with N hops). Always the same underlying tension: shared medium means more users/hops = less per-user throughput.

Energy as a First-Class Constraint

BLE's low data rate is an energy tradeoff. WSN's entire design philosophy is energy-first. Even cellular power control (adjusting transmit power by 1 dB steps) is fundamentally about minimizing energy/interference while maintaining quality.

"No Contemporaneous Path" Shows Up Twice

DTN's entire architecture is built around this idea, and VANET's "unidirectional links" challenge is the same idea in miniature — a connection window so brief or asymmetric that a normal two-way handshake can't be assumed to complete.

Mesh vs. Ad-Hoc Is the Single Most-Tested Distinction

The deck spends three full slides explicitly contrasting WMN and WAHN side-by-side — that level of dedicated comparison space is a strong signal the professor considers this distinction exam-critical. The one-line test: does this network have nodes whose ONLY job is to relay (mesh) or does every node play an identical role (ad-hoc)?
